# /pr

현재 브랜치의 변경사항을 분석해서 PR description을 작성하고 GitHub PR을 생성한다.

## Step 1: 변경사항 파악

다음을 순서대로 실행한다:

1. `git branch --show-current` — 현재 브랜치명 확인
2. `git log main..HEAD --oneline` — main과의 커밋 차이
3. `git diff main...HEAD --stat` — 변경된 파일 목록
4. 주요 변경 파일들을 읽어서 구현 내용 파악

## Step 2: PR description 작성

아래 형식으로 작성한다.

```markdown
## Overview
[1-3줄로 이번 PR의 핵심을 설명. "무엇을" 구현했는지]

## 구현 내용
[주요 변경사항을 항목별로. 파일명/훅명/컴포넌트명 구체적으로]

## 주요 결정사항
[기술적 선택이나 트레이드오프가 있으면 이유와 함께 설명. 없으면 생략]

## Test plan
- [ ] [테스트 항목 1]
- [ ] [테스트 항목 2]
- [ ] [엣지케이스]

## 스크린샷
[UI 변경이 있으면 "스크린샷 첨부 필요" 메모. 없으면 생략]
```

## Step 3: PR 생성

다음을 확인하고 실행:

1. 원격 브랜치에 push 되어 있는지 확인 (`git status`)
2. push 안 되어 있으면 `git push -u origin {브랜치명}` 실행
3. `gh pr create` 실행 (main 브랜치 대상)

## 금지사항

- PR body에 "Generated by Claude Code", "Co-Authored-By: Claude" 등 AI attribution 금지
- 제목에 브랜치명 그대로 복붙 금지 (의미 있는 제목으로)
- 50자 초과 PR 제목 금지

## PR 제목 규칙

브랜치명 `feat/swap-page` → PR 제목 `feat: 토큰 스왑 페이지 구현`  
브랜치명 `fix/token-balance-null` → PR 제목 `fix: 토큰 잔고 null 처리 오류 수정`
