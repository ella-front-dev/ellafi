# 슬래시 커맨드 만드는 법

## 구조

`.claude/commands/{커맨드명}.md` 파일 하나 = `/커맨드명` 하나.

```
.claude/commands/commit.md  →  터미널에서 /commit 입력
.claude/commands/pr.md      →  터미널에서 /pr 입력
```

## 파일 작성법

커맨드 파일 = Claude에게 줄 프롬프트.  
"이 커맨드 실행되면 이렇게 행동해라"를 서술.

```markdown
# /커맨드명

## 목적
무엇을 하는 커맨드인지 한 줄 설명

## 실행 방법
Step별로 무엇을 해야 하는지

## 출력 형식
결과물을 어떤 형태로 보여줄지

## 제약사항
하면 안 되는 것
```

## 인자 전달

`$ARGUMENTS` 플레이스홀더로 커맨드 인자를 받음.

```
사용자 입력: /review src/hooks/use-token-prices.ts
커맨드 파일에서: $ARGUMENTS = "src/hooks/use-token-prices.ts"
```

```markdown
# /review

대상 파일: $ARGUMENTS

## Step 1: 파일 읽기
$ARGUMENTS 경로의 파일을 읽어라.

## Step 2: 체크리스트
- TypeScript strict 위반 여부
- naming convention 위반 여부
- ...
```

## 서브디렉토리 구조

커맨드가 많아지면 네임스페이스로 분리.

```
.claude/commands/
  commit.md         →  /commit
  pr.md             →  /pr
  review.md         →  /review
  ellafi/
    phase-check.md  →  /ellafi:phase-check
```

## 현재 Ellafi 커맨드

| 파일 | 커맨드 | 용도 |
|------|--------|------|
| `commands/commit.md` | `/commit` | git diff 분석 → 커밋 메시지 생성 |
| `commands/pr.md` | `/pr` | 구현 내용 분석 → PR description 작성 |

## 커맨드 작성 팁

### 구체적인 출력 형식 명시

```markdown
## 출력 형식

아래 형식으로만 출력해라:

```
타입(범위): 제목

- 변경 내용 1
- 변경 내용 2
```
```

### 금지사항 명시

```markdown
## 금지사항
- "Co-Authored-By: Claude" 같은 AI attribution 문구 포함 금지
- PR body에 "Generated by Claude Code" 문구 금지
- 영어 커밋 금지 (한국어만)
```

### 기존 파일 참조

```markdown
## 참조
커밋 컨벤션은 최근 git log 5개를 참고해서 스타일 맞춰라.
```
